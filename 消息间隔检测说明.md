# 消息间隔检测功能说明

## 功能概述

消息间隔检测是一个高级的反机器人刷屏功能，通过分析玩家发送消息的时间间隔模式来识别自动化脚本或机器人行为。

## 工作原理

### 第一阶段：模式检测（初次触发）

系统会记录每个玩家最近3条消息的发送间隔。当检测到连续3条消息的间隔几乎完全一致时（误差在1-2ms内），系统会：

1. 识别这是一个可疑的机器人模式
2. 计算平均间隔时间
3. 将该玩家标记为"疑似机器人"
4. 记录预期的间隔模式

**示例：**
```
消息1 -> 消息2: 1000ms
消息2 -> 消息3: 1001ms
消息3 -> 消息4: 1000ms

结果：检测到机器人模式（平均间隔1000ms，误差≤2ms）
```

### 第二阶段：持续拦截（标记后）

一旦玩家被标记为"疑似机器人"，系统会：

1. 持续监控该玩家的消息间隔
2. 如果后续消息继续保持相同的间隔模式（误差≤2ms），立即拦截
3. 直到玩家改变发送模式或发送成功的消息

**示例：**
```
玩家已被标记（预期间隔：1000ms）

消息5间隔: 1001ms -> 拦截（匹配模式）
消息6间隔: 999ms  -> 拦截（匹配模式）
消息7间隔: 1500ms -> 通过（模式改变，自动解除标记）
```

## 解除标记的条件

玩家的"疑似机器人"标记会在以下情况下自动解除：

1. **发送间隔改变**：当消息间隔与预期模式差异超过2ms时
2. **成功发送消息**：当一条消息通过所有检测并成功发送时
3. **玩家离线**：玩家下线后，所有数据会被清理

## 配置选项

### 启用/禁用

```bash
# 启用消息间隔检测
/chat antispam toggle interval

# 查看当前状态
/chat antispam status
```

### 配置文件

在 `config.yml` 中：

```yaml
anti-spam:
  interval-check: true  # true=启用，false=禁用
```

## 检测参数

- **间隔容差**：2ms（硬编码）
- **检测窗口**：最近3条消息
- **标记持续时间**：直到模式改变或成功发送消息

## 违规类型

当检测到机器人模式时，会记录违规类型为：`bot-pattern`

可以通过以下命令查看：
```bash
/chat stats <玩家名> list
```

## 日志输出

### 初次检测日志
```
[INFO]: Initial bot pattern detected for player <UUID>: 1000ms, 1001ms, 1000ms (avg=1000ms, tolerance=2ms)
[INFO]: Player flagged - will block all subsequent messages with similar intervals
```

### 持续拦截日志
```
[INFO]: Flagged player <UUID> continues bot pattern: 1001ms (expected ~1000ms)
```

### 解除标记日志
```
[INFO]: Player <UUID> changed interval pattern, unflagging
[INFO]: Player <UUID> sent successful message, clearing bot flag
```

## 使用场景

### 适用于检测：

1. **自动化脚本**：使用固定延迟的刷屏脚本
2. **宏程序**：键盘宏或鼠标宏
3. **机器人客户端**：使用定时器发送消息的机器人
4. **作弊客户端**：带有自动聊天功能的作弊客户端

### 不会误判：

1. **正常玩家**：人类玩家的输入间隔会有自然的波动
2. **快速打字**：即使打字很快，间隔也不会精确到毫秒级一致
3. **复制粘贴**：偶尔的快速发送不会触发（需要连续3次）

## 与其他检测的配合

消息间隔检测与其他反垃圾功能协同工作：

1. **基础延迟检测**：先检查是否违反最小间隔（如3秒）
2. **间隔模式检测**：检测是否为机器人行为
3. **相似度检测**：检测消息内容是否重复
4. **违规词过滤**：检测是否包含违规词

## 性能影响

- **内存占用**：每个玩家约占用 ~200 bytes（存储3个间隔值 + 标记信息）
- **CPU占用**：极低（仅简单的数学比较）
- **自动清理**：离线玩家数据每10分钟自动清理

## 测试建议

### 测试机器人检测

1. 使用脚本或宏以固定间隔（如1秒）发送3条消息
2. 第3条消息应该被拦截并显示"检测到异常发送模式（疑似机器人）"
3. 继续以相同间隔发送，所有消息都会被拦截
4. 改变发送间隔或手动输入消息，标记会被解除

### 测试正常玩家

1. 正常手动输入并发送多条消息
2. 即使发送很快，也不应该触发机器人检测
3. 只有在间隔精确到毫秒级一致时才会触发

## 注意事项

1. **白名单玩家**：白名单玩家仍会记录间隔数据，但不会被拦截
2. **禁言玩家**：禁言检测优先级高于间隔检测
3. **私信检测**：私信（/w, /msg等）也会进行间隔检测
4. **数据持久化**：标记信息不会保存到文件，重启服务器后清空

## 命令总结

```bash
# 启用/禁用间隔检测
/chat antispam toggle interval

# 查看配置状态
/chat antispam status

# 查看玩家违规记录（包括bot-pattern）

# 重新加载配置

```
