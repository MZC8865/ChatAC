# 自动禁言功能说明

## 功能概述

自动禁言功能会自动监控玩家的违规行为，当玩家在短时间内频繁违规时，系统会自动对其进行禁言处罚。

## 工作原理

### 违规计数

系统会记录每个玩家的所有违规行为，包括：
- 刷屏（spam）
- 相似消息（similarity）
- 违规词（profanity）
- 机器人模式（bot-pattern）
- 禁言期间发言（muted）

### 触发条件

当玩家在**1分钟内**的违规次数达到或超过设定的阈值时，系统会自动触发禁言。

**默认配置：**
- 阈值：5次违规/分钟
- 禁言时长：300秒（5分钟）

### 触发流程

1. 玩家发送消息触发违规检测
2. 系统记录违规并检查最近1分钟的违规次数
3. 如果违规次数 ≥ 阈值，触发自动禁言
4. 玩家被禁言指定时长
5. 玩家收到禁言通知

## 配置选项

### 配置文件

在 `config.yml` 中：

```yaml
anti-spam:
  auto-mute:
    enabled: true                 # 是否启用自动禁言
    violations-per-minute: 5      # 每分钟违规次数阈值
    mute-duration: 300            # 禁言时长（秒），默认5分钟
```

### 命令配置

#### 启用/禁用自动禁言
```bash
/chat antispam toggle automute
```

#### 设置违规阈值
```bash
# 设置为每分钟3次违规触发
/chat antispam automute threshold 3

# 设置为每分钟10次违规触发
/chat antispam automute threshold 10
```

#### 设置禁言时长
```bash
# 设置为30秒
/chat antispam automute duration 30s

# 设置为5分钟
/chat antispam automute duration 5m

# 设置为1小时
/chat antispam automute duration 1h
```

支持的时间格式：
- `s` 或 `秒` - 秒
- `m` 或 `分` 或 `分钟` - 分钟
- `h` 或 `时` 或 `小时` - 小时
- `d` 或 `天` - 天

#### 查看当前配置
```bash
/chat antispam status
```

输出示例：
```
AntiSpam Status:
- autoMute: 启用 (阈值: 5 次/分钟, 时长: 300 秒)
```

## 特殊情况处理

### 已被禁言的玩家

如果玩家已经被禁言（手动禁言或之前的自动禁言），不会重复触发自动禁言。

### 白名单玩家

白名单玩家不会触发违规检测，因此也不会被自动禁言。

### 违规记录清理

违规记录会随着时间推移自动过期：
- 只统计最近1分钟的违规
- 超过1分钟的违规不计入阈值
- 玩家离线后，违规记录会在下次清理任务时被清除

## 配置建议

### 严格模式（适合公共服务器）
```yaml
auto-mute:
  enabled: true
  violations-per-minute: 3      # 3次违规即禁言
  mute-duration: 600            # 禁言10分钟
```

### 宽松模式（适合小型服务器）
```yaml
auto-mute:
  enabled: true
  violations-per-minute: 10     # 10次违规才禁言
  mute-duration: 180            # 禁言3分钟
```

### 测试模式（用于调试）
```yaml
auto-mute:
  enabled: true
  violations-per-minute: 2      # 2次违规即触发
  mute-duration: 30             # 禁言30秒
```

## 与手动禁言的关系

### 优先级

自动禁言和手动禁言使用相同的禁言系统，但：
- 手动禁言可以覆盖自动禁言
- 自动禁言不会覆盖已存在的禁言

### 解除禁言

管理员可以使用以下命令解除自动禁言：
```bash
/chat unban <玩家名>
```

## 权限要求

- 配置自动禁言：`antispam.admin`
- 手动解除禁言：`antispam.admin`

## 注意事项

1. **时间窗口**：违规计数基于1分钟滚动窗口，不是固定的每分钟重置
2. **累积效果**：所有类型的违规都会累积计数
3. **不可叠加**：已被禁言的玩家不会再次触发自动禁言
4. **配置持久化**：通过命令修改的配置会自动保存到配置文件
5. **重启生效**：修改配置文件后需要重启服务器或使用 `/chat antispam reload`

## 测试建议

### 测试自动禁言触发

1. 设置较低的阈值：`/chat antispam automute threshold 2`
2. 设置较短的禁言时长：`/chat antispam automute duration 30s`
3. 快速发送3条消息触发刷屏检测
4. 观察是否在第2次违规后触发自动禁言
5. 等待30秒后确认禁言自动解除

### 测试配置修改

1. 查看当前配置：`/chat antispam status`
2. 修改阈值：`/chat antispam automute threshold 10`
3. 再次查看配置确认修改成功
4. 重启服务器后确认配置持久化
