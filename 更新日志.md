# ChatAC 更新日志

## 版本 3.1 - 2026-04-24

### 新增功能

#### 1. 统计命令彩色输出
- `/chat stats <玩家>` 命令现在使用彩色输出，更易读
  - 标题：绿色
  - 字段名：青色
  - 数值：白色
  - 违规率根据严重程度显示不同颜色：
    - 绿色：< 20%
    - 黄色：20% - 50%
    - 红色：> 50%

#### 2. 违规记录查询命令
- 新增 `/chat stats <玩家> list [时间]` 命令
  - 显示指定时间内的所有违规记录
  - 默认时间：5分钟（如果不指定）
  - 支持时间格式：`30s`, `5m`, `1h`, `1d`
  - 彩色输出：
    - 时间戳：灰色
    - 违规类型：红色
    - 违规消息：黄色
    - 重复次数：金色
  - 自动合并连续相同的违规记录，显示次数

#### 3. 违规词检测增强
- 添加字符标准化处理，防止绕过检测
- 自动移除空格、符号和特殊字符
- 只保留字母和数字进行匹配
- 示例：
  - "傻 逼" → 检测为 "傻逼"
  - "傻@逼" → 检测为 "傻逼"
  - "s h i t" → 检测为 "shit"

### 技术改进

#### ProfanityFilter.java
- 新增 `normalizeString()` 方法
  - 移除所有空格、符号和特殊字符
  - 转换为小写
  - 只保留字母和数字
- 更新 `containsProfanity()` 和 `findBlockedWord()` 方法使用标准化
- 违规词列表在加载时自动标准化

#### AntiSpamCommand.java
- 增强 `/chat stats` 命令的输出格式
- 新增 `handleStatsListCommand()` 方法处理违规记录查询
- 新增 `groupViolations()` 方法合并连续相同违规
- 新增 `GroupedViolation` 内部类存储分组信息
- 完善 tab 补全支持 `list` 子命令和时间参数

#### StatsManager.java
- `ViolationRecord` 类已包含时间戳、类型和消息内容
- `getRecentViolations()` 方法支持按时间范围查询
- 自动保存最近100条违规记录

### 使用示例

```
# 查看玩家统计（彩色输出）
/chat stats maozhichen

# 查看最近5分钟的违规记录（默认）
/chat stats maozhichen list

# 查看最近30分钟的违规记录
/chat stats maozhichen list 30m

# 查看最近1小时的违规记录
/chat stats maozhichen list 1h
```

### 违规记录输出示例

```
=== maozhichen 的违规记录（过去 5 分钟）===
共 15 条违规，3 组记录：
[19:45:23] spam: 测试消息 (*8)
[19:46:15 - 19:46:45] profanity: 傻逼 (*5)
[19:47:30] similarity: 重复消息 (*2)
```

### 注意事项

1. 违规词标准化是自动的，无需手动配置
2. 旧的违规词列表会在插件重载时自动标准化
3. 统计数据保存在 `stats.yml`，每5分钟自动保存
4. 最多保存每个玩家最近100条违规记录
