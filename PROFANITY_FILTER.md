# 违规词过滤功能说明

## 功能概述

违规词过滤功能可以自动检测并拦截包含违规词汇的聊天消息，保持服务器聊天环境的健康。

## 特性

- ✅ 自动检测消息中的违规词
- ✅ 支持中文和英文违规词
- ✅ 不区分大小写（"FUCK" 和 "fuck" 都会被检测）
- ✅ 可配置的违规词列表
- ✅ 可以随时开关过滤功能
- ✅ 优先级最高（在其他检测之前执行）

## 配置方法

### 1. 编辑配置文件

编辑 `plugins/AntiSpam/config.yml`：

```yaml
anti-spam:
  profanity-filter: true          # 是否启用违规词过滤
  blocked-words:                  # 违规词列表
    - "傻逼"
    - "fuck"
    - "shit"
    - "草泥马"
    - "sb"
    - "nmsl"
    - "cnm"
    - "傻B"
    - "煞笔"
    # 添加更多违规词...
```

### 2. 添加违规词

在 `blocked-words` 列表中添加需要屏蔽的词汇：

```yaml
blocked-words:
  - "你的违规词1"
  - "你的违规词2"
  - "your_bad_word"
```

### 3. 重新加载配置

修改配置文件后，使用命令重新加载：

```
/chat antispam reload
```

## 使用命令

### 开关违规词过滤

```bash
/chat antispam toggle profanity
```

这个命令会：
- 如果当前是启用状态 → 禁用过滤
- 如果当前是禁用状态 → 启用过滤
- 自动保存到配置文件

### 查看状态

```bash
/chat antispam status
```

输出示例：
```
AntiSpam Status:
- delayMs: 20 ms
- similarityCheck: 启用
- similarityThreshold: 80.00%
- minLengthForCheck: 3
- profanityFilter: 启用
- blockedWords: 6 个
```

## 工作原理

### 检测优先级

违规词检测是最高优先级，检测顺序为：

1. **违规词检测** ← 最先执行
2. 时间间隔检测
3. 相似度检测

这意味着即使消息通过了时间间隔检测，只要包含违规词就会被拦截。

### 检测方式

- 不区分大小写：`"FUCK"`, `"fuck"`, `"FuCk"` 都会被检测
- 包含匹配：只要消息中包含违规词就会被拦截
  - 例如：违规词是 `"sb"`，消息 `"你个sb"` 会被拦截

### 拦截提示

当消息被拦截时：
- 玩家看到：`有违规文字：<具体违规词>`（红色文字 + 黄色违规词）
- 服务器日志：`[AntiSpam] Blocked chat from PlayerName (profanity detected: 违规词)`

## 注意事项

### 1. 误判问题

某些违规词可能会导致误判，例如：
- 违规词 `"sb"` 可能会拦截正常的英文单词
- 违规词 `"日"` 可能会拦截日期相关的消息

建议：
- 仔细选择违规词
- 避免使用过短或过于常见的词汇
- 定期检查日志，调整违规词列表

### 2. 性能考虑

- 违规词列表不宜过长（建议 < 100 个）
- 每条消息都会检查所有违规词
- 如果列表过长可能影响性能

### 3. 绕过检测

玩家可能尝试通过以下方式绕过检测：
- 添加空格：`"傻 逼"`
- 使用符号：`"傻@逼"`
- 使用拼音：`"shabi"`

建议：
- 将常见变体也加入违规词列表
- 定期更新违规词列表

## 管理建议

### 1. 定期审查

定期检查服务器日志，查看被拦截的消息：

```
[AntiSpam] Blocked chat from PlayerName (profanity detected: xxx)
```

根据日志调整违规词列表。

### 2. 分级管理

可以考虑：
- 轻度违规词：仅拦截消息
- 重度违规词：拦截消息 + 警告玩家
- 严重违规词：拦截消息 + 临时禁言

（后两项需要额外开发）

### 3. 玩家反馈

如果玩家反馈正常消息被误拦截：
1. 检查日志确认被拦截的词
2. 评估是否需要从列表中移除
3. 使用 `/chat antispam reload` 重新加载配置

## 故障排除

### 问题：违规词没有被拦截

1. 检查过滤是否启用：
   ```
   /chat antispam status
   ```

2. 检查配置文件格式是否正确

3. 确认已重新加载配置：
   ```
   /chat antispam reload
   ```

### 问题：正常消息被误拦截

1. 查看日志确认被拦截的词
2. 从配置文件中移除该词
3. 重新加载配置

### 问题：违规词列表为空

检查配置文件中 `blocked-words` 的格式：

```yaml
# 正确格式
blocked-words:
  - "word1"
  - "word2"

# 错误格式
blocked-words: []
```
